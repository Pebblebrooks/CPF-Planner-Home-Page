CPF Planner Pro - Landing Page
Welcome to the repository for the CPF Planner Pro promotional landing page. This is a lightweight, high-performance static marketing website designed to introduce users to the CPF Planner Pro web application.
🔗 Live Links
•	Marketing Site (Front Door): https://cpfplannerhome.netlify.app/
•	Main Web Application: https://cpfplannerv3.netlify.app/
📖 Overview
To ensure the best performance and maintain a clean build process, the CPF Planner Pro ecosystem is split into two parts:
1.	This Repository (index.html): A static HTML/CSS/JS landing page optimized for SEO, speed, and marketing. It funnels users to the main tool.
2.	The Application Repository: A React/Vite web application that handles the complex financial calculations and interactive dashboard.
✨ Features
This landing page is built with pure HTML, CSS, and Vanilla JavaScript (no heavy frameworks required) and includes:
•	Fully Responsive Design: Adapts flawlessly to Desktop, Tablet, and Mobile devices.
•	Theme Toggling: Built-in Light/Dark mode that respects system preferences and saves user choice via localStorage.
•	Custom Smooth Scrolling: A mathematically calculated smooth scroll routing system that bypasses mobile browser bugs.
•	Embedded Video Player: Seamless, inline YouTube iframe integration for tool demonstrations.
•	Viewport Testing Tool: A built-in dropdown to force Desktop (FHD/HD) constraints for testing purposes.
•	CSS Animations: Intersection Observers trigger smooth fade-in animations as the user scrolls down the page.
🚀 Deployment
This site is designed to be hosted seamlessly on Netlify as a static site.
Manual Deployment (Drag and Drop):
1.	Download or clone this repository so you have the index.html file in a folder.
2.	Log in to Netlify.
3.	Go to Sites -> Add new site -> Deploy manually.
4.	Drag and drop the folder containing index.html into the upload area.
GitHub Integration Deployment:
1.	Connect this GitHub repository to Netlify.
2.	Netlify will automatically detect the index.html file at the root.
3.	No build command or publish directory is required.
